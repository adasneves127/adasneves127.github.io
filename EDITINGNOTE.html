Here is a quick rundown of what these "<" & ">" things mean.

"<" & ">"'s represent "tags." These tags denote how objects should be rendered while viewing. Different tags control diferent looks.

A good analogy for this layout can be like a Word Document.

We will start analyzing these from the top down. Any tag that has a slash (/) in it denotes a closing tag, which represents the end of its corresponding opening tag.

  • <!DOCTYPE html> -- This denotes which version of the HTML standard we are using. In this case, we are using just normal HTML.
  • <html lang="en"> -- This tells the website that this is the start of the website, and that the text will all be English.
  • <head> -- This is the heading of a website. It contains information about the website, as well as anything we bring in from outside this file.
  • We are going to ignore the <meta> tags, as they contain specifics to tell the web browser what to expect...
  • <title> -- This tag tells the web browser what to put on the tab of the webpage.
  • <link> -- This is used to import our styling from other files. In this case, it saves me from putting the same code into each document.
  • </head> -- This is used to tell the browser that it is the end of the heading of our website.
  • <body> -- The real meat & potatoes of our website will be within the body tag.
  • <div> -- A division within the website, to which styles can be applied to.
  • class -- This is another way to denote which styles get applied where. Styles can be applied to Tags, or classes.
  • <h1> - <h6> -- These are different headers. Headers contain information that is usually considered more important, and thus, is rendered differently.
  • <ul> -- This is an Unordered list. Rather than have numbers to show the start of each line, it will instead use bullet points.
  • <ol> -- This is an Ordered list. Each line item within this will start the line with a number instead.
  • <li> -- This is a Line Item, which is used to denote the start of a new element in our list.
  • <style> -- This denotes new styling rules, not contained within our external file.
  • <script> -- This denotes new scripting code to be run. This could include updating text on screen, and much more!
  • <br> -- Line break, kinda like hitting the enter button.
  • <hr> -- Horrizontal Rule, which will insert a solid line horrizontally across the screen.

In some of these recipes, the ingredients will be in a table instead. The layout of a table is as such:

  • <table> -- Everything that falls between this tag, and it's corresponding closing tag will be rendered as a table.
  • <tr> -- This is a Table Row.
  • <td> -- This will be data held within the table.
  • <th> -- This is a heading for a table's column.

To edit code for the website, once you are on the page for the recipe, press the button that looks like a pencil. If this button is grayed out, it means that you are not logged into this website.
If you are not logged in, either log into my account, or create your own.

Once you hit the pencil button, it will throw you into the editor for the code. In this window, scroll to the line that you want to edit, and type normally, ignoring the tags.

If you need to insert a tag, Observe what tags have been used for similar items, and use that one. If you are unsure, please ask me!

Once you are done, scroll to the bottom of the page to where it says "Commit changes"
It will ask for a breif description of what you have done to the page. Put a brief description such as:

Updated Instructions for Clarity.

Once your description is entered, press the button that says "Commit changes"
Please note that it will not update the website immediately. It should update within 3-5 minutes of the change being made.
